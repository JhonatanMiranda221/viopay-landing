💳 VioPay - Interface Financeira Global
O VioPay é um projeto de interface para uma fintech focada em viajantes internacionais, oferecendo soluções de câmbio, cartões globais e um checklist inteligente de embarque. Este projeto foi desenvolvido com o objetivo de consolidar fundamentos de arquitetura front-end, priorizando a semântica, a organização e as boas práticas de desenvolvimento moderno.

🎯 Objetivo do Projeto
Diferente de projetos focados em bibliotecas e frameworks, o VioPay foi construído utilizando Vanilla Tech Stack (HTML, CSS e JS puros). O foco principal foi o domínio da "arquitetura do pensamento" no desenvolvimento web, garantindo que a base técnica seja sólida antes da transição para ferramentas de abstração.

🛠️ Tecnologias e Conceitos Aplicados
Front-End
HTML5 Semântico: Uso rigoroso de tags estruturais (<nav>, <main>, <section>, <article>) para garantir acessibilidade e otimização para motores de busca (SEO).

CSS3 Moderno: * Metodologia BEM (Block Element Modifier): Organização de classes para garantir manutenibilidade e evitar conflitos de especificidade.

CSS Variables (Custom Properties): Centralização da identidade visual no :root para facilitar alterações globais (Single Source of Truth).

Flexbox Layout: Utilizado para alinhamentos dinâmicos, distribuição de espaço e centralização de componentes complexos.

Box Model Mastery: Controle preciso de padding, margin e border com uso de box-sizing: border-box.

JavaScript (Lógica de Dados): * Tratamento de dados estruturados em objetos para simular respostas de API.

Manipulação dinâmica do DOM para filtragem de conteúdos.

Mentalidade Back-End no Front-End
Contratos de Dados: Planejamento da estrutura de objetos antes da implementação visual.

Separação de Preocupações (SoC): Divisão clara entre a estrutura (HTML), a estética (CSS) e o comportamento (JS).

Padronização: Implementação de um Reset CSS e variáveis globais, simulando uma infraestrutura de ambiente configurável.

🚀 Principais Funcionalidades
Landing Page Responsiva: Design otimizado para diferentes resoluções.

Hero Section de Impacto: Chamada para ação (CTA) com hierarquia visual clara e elementos de prova social.

Navbar Inteligente: Menu fixo com estados de interação (hover e active).

Checklist Dinâmico (Em desenvolvimento): Filtros por destino (Europa/China) que alteram os dados exibidos em tempo real.

📚 Aprendizados Adquiridos
Durante a construção deste projeto, foram explorados conceitos de:

Hierarquia Visual: Como guiar o olhar do usuário através de pesos de fonte e contrastes de cores.

Fluxo do DOM: O entendimento de como o navegador renderiza e organiza as "caixas" na tela.

Estados de Interface: Implementação de feedbacks visuais (hover, active, transition) para melhorar a experiência do usuário (UX).
